# 外贸客户自动开发工作流模板

## 1. 工作流目标

让 Skill获客平台按固定步骤自动完成外贸客户开发前半段：

```text
输入产品和市场
-> 自动生成搜索任务
-> 找公司
-> 整理官网和公司信息
-> 核验 WhatsApp / 电话 / 邮箱 / LinkedIn
-> AI 评分
-> 生成开发话术
-> 放入 CRM 跟进池
-> 导出表格
```

第一版建议做成“自动整理 + 人工确认发送”，不要一开始自动群发。

## 2. 触发方式

### 手动触发

用户在平台输入：

- 产品
- 目标国家
- 客户类型
- 数量
- 是否 WhatsApp 优先
- 是否需要邮箱
- 是否需要 LinkedIn

点击：`开始找客户`

### 定时触发

每天或每周自动跑一次，例如：

```text
每天 09:00
产品：全屋定制、橱柜、衣柜、木门、家具出口
国家：澳洲
客户类型：Builder + Designer + Kitchen Company + Importer
数量：50
联系方式优先级：WhatsApp > 电话 > 邮箱 > LinkedIn
```

## 3. 输入模板

```yaml
task_name: 澳洲全屋定制客户开发
product: 全屋定制、橱柜、衣柜、木门、家具出口
target_countries:

export_format:
  - csv
  - xlsx
```

## 4. 自动流程

### Step 1: 创建找客任务

系统创建一条 `lead_task`：

- 任务名
- 产品
- 国家
- 客户类型
- 数量
- 状态：`Running`

输出：

```text
任务创建成功，进入客户发现阶段。
```

### Step 2: 生成搜索关键词

AI 根据产品和客户类型生成英文关键词：

```text
custom cabinetry Australia builder
kitchen company Australia custom cabinets
wardrobe company Australia contact
wood doors Australia builder supplier
kitchen importer Australia cabinets
joinery company Australia kitchen wardrobe
```

输出：

- search_keywords
- source_plan

### Step 3: 发现客户公司

系统从公开商业来源收集公司：

- Google 搜索结果
- 公司官网
- LinkedIn 公司页
- 行业目录
- 展会目录
- 协会目录
- B2B 平台公开页

每条公司必须保留：

- 公司名
- 官网
- 国家
- 来源链接
- 初步客户类型

输出：

```text
candidate_companies[]
```

### Step 4: 补全公司信息

对每家公司整理：

- 公司简介
- 主营产品
- 客户类型
- 产品匹配点
- 官网摘要
- 城市/地区
- 来源证据

AI 输出中文摘要：

```text
该公司是澳洲厨房和衣柜设计公司，覆盖厨房、衣柜、洗衣房等空间，适合开发全屋定制和橱柜项目客户。
```

### Step 5: 核验联系方式

联系方式按优先级处理：

1. WhatsApp
2. 电话/手机
3. 邮箱
4. LinkedIn
5. 官网联系表单

WhatsApp 状态：

```text
已验证WhatsApp
电话可核验WhatsApp
未公开WhatsApp
```

邮箱状态：

```text
官网邮箱
公开商业邮箱
联系表单
未找到
```

重要规则：

- 不编造 WhatsApp。
- 不把所有电话自动当成 WhatsApp。
- 不猜邮箱，除非单独标注为“猜测邮箱”。
- 每个联系方式都必须有来源或状态说明。

### Step 6: AI 客户评分

评分满分 100：

| 评分项 | 分数 |
|---|---:|
| 产品匹配度 | 30 |
| 客户类型匹配度 | 20 |
| 市场匹配度 | 15 |
| 联系方式完整度 | 15 |
| 来源可信度 | 10 |
| 商业潜力 | 10 |

优先级：

```text
A: 85-100，优先开发
B: 75-84，值得开发
C: 60-74，二次筛选
D: 60以下，低优先级
```

### Step 7: 生成开发内容

每个客户生成：

- 英文开发信
- WhatsApp 开场白
- LinkedIn 私信
- 跟进邮件 1
- 跟进邮件 2

开发信必须引用客户真实信息：

```text
I noticed that {Company} works with {specific category}.
```

不能写：

```text
We know you are buying cabinets now.
```

除非来源明确证明。

### Step 8: 进入 CRM

每个客户进入 CRM：

```text
New -> Verified -> Contacted -> Replied -> Quoting -> Sample -> Negotiating -> Won
```

其他状态：

```text
Lost
Not Fit
```

默认规则：

- A/B 级客户：进入 `Verified`
- C 级客户：进入 `New`
- D 级客户：进入 `Not Fit`

### Step 9: 导出结果

导出字段：

```text
公司名称
国家
城市
官网
客户类型
主营产品
产品匹配点
来源链接
WhatsApp状态
WhatsApp/电话
邮箱状态
邮箱
LinkedIn
联系人
职位
AI评分
优先级
AI分析
首封开发信
WhatsApp开场白
CRM状态
下次跟进时间
备注
```

## 5. 自动运行规则

### 成功条件

任务完成需要满足：

- 客户数量达到目标数量的 80% 以上。
- 每个客户至少有官网或可信来源链接。
- 每个客户有评分。
- A/B 级客户有开发信和 WhatsApp 开场白。
- 联系方式没有编造数据。

### 失败处理

如果客户数量不足：

```text
扩大关键词
扩大相邻客户类型
加入行业目录和展会目录
保留不足原因
```

如果联系方式不足：

```text
标记为 Contact form only 或 Not found
进入人工补全队列
不编造联系方式
```

## 6. 人工确认节点

自动流程跑完后，系统必须停在人工确认节点：

```text
请确认 A/B 级客户是否进入联系队列。
请确认开发信和 WhatsApp 话术是否需要修改。
确认后再发送或复制使用。
```

不建议第一版自动发送消息。

## 7. 每日工作节奏

```text
09:00 自动跑新客户任务
10:00 人工检查 A/B 级客户
11:00 复制 WhatsApp / 邮件开发
15:00 更新 CRM 状态
17:30 查看回复和设置明日跟进
```
